02   CfC + Attention Roles
Why CfC stays in the temporal branch
這條主線不是把 Transformer 與 CfC 二選一，而是分工：
Transformer / attention 負責 token 間的讀取與訊息交換；
CfC 負責處理不規則時間步長下的連續時間演化。

目前 CfC 仍然是 closed-form recurrent update，而非數值 ODE solver。
CfC forward
xh = cat([x, h], dim=−1)
f1 = tanh(W₁ · xh + b₁)
f2 = tanh(W₂ · xh + b₂)
τ_a = exp(log_tau_a)
t_b = W_b · xh + b_b
gate = σ(−τ_a · Δt + t_b)
h_new = gate · f1 + (1 − gate) · f2
Who does what in the current model
Token self-attention先讓 sensors 交換空間上下文
Temporal CfC對每個 token 做 causal time evolution
Decoder cross-attention讓 query 自己決定該讀哪些 branch tokens
Sparse-data design rule
訓練主線只允許使用：
1. 真實量到的 sensor data (`u,v,omega`)
2. momentum residual
3. continuity residual
4. causal / time-marching training rule

不把 vorticity / spectrum / energy / enstrophy 放進 training loss；
這些量只保留在 evaluation 作診斷工具。
No single-vector branch
目前主線已放棄「先壓成單一 h_enc 再解碼」，改成 sensor-level branch tokens 直接被 decoder 讀取。
Diagnostics only
vorticity / spectrum / kinetic energy / enstrophy 仍然保留，但只在 evaluation 階段使用，不再作 dense supervision。
03   Sparse-Data Training Loop
Sensor Data Loss
1
Encode sensor history
sensor_vals + sensor_pos 先形成 thin sensor tokens，再經 token self-attention 與 temporal CfC 得到 causal branch tokens。
2
Sample sparse supervised queries
只在實際量到的 sparse observations 上做點值監督；目前觀測通道是 u, v, omega，不依賴 full-field gradient labels。
num_query_points = 512
3
Decode from branch tokens
query trunk feature 對 branch tokens 做 cross-attention，再經 branch/trunk fusion 輸出 latent field u / v / p。
4
Pointwise sensor/data consistency
L_data 只對真實量測通道做 MSE；若觀測是 omega，則由模型的 u,v 導數現算，不假設 sensor 直接提供 p。
data_loss_weight = 1.0
Physics Residual
1
Sample collocation pts
在目前開放的 time-marching 時窗內，隨機取 (x,y,t) 物理點，並開啟 requires_grad=True。
num_physics_points = 128
2
Predict u, v, p
透過同一個 branch token 記憶與 query decoder，分別求出 u, v, p 在 collocation points 的值。
3
Autograd PDE residuals
用 torch.autograd.grad 計算時間導數、空間一階與二階導數，形成 unsteady incompressible NS residual。
4
NS residual + continuity
L_phys = momentum residual + λ·continuity residual。這是 sparse-data 主線可直接信任的物理 supervision。
physics_loss_weight = 0.01
Optimization & Diagnostics
1
Total loss
主線只用 L = λ_d · L_data + λ_p · L_phys，不再混入 vorticity / spectrum / dense auxiliary losses。
2
Time marching + physics ramp
先限制可見時間窗，再逐步開放整段時域；同時讓 physics weight 緩慢介入，避免 early collapse。
learning_rate = 1e-3
3
Diagnostics stay outside training
vorticity / spectrum / kinetic energy / enstrophy 全部保留在 evaluation，作為場品質與物理結構診斷，而不是 supervision。
4
Stable optimizer path
AdamW + cosine LR + gradient clipping；目前目標是讓 sparse-data 主線先穩定、不 collapse，再談更強結構對齊。
max_grad_norm = 1.0
Sparse-Data Objective
L = λ_d · L_data  +  λ_p · L_NS  +  λ_c · L_continuity
L_NS = |∂u/∂t + u·∂u/∂x + v·∂u/∂y + ∂p/∂x − ν·∇²u − f_x|²  +  |∂v/∂t + u·∂v/∂x + v·∂v/∂y + ∂p/∂y − ν·∇²v|²
L_continuity = |∂u/∂x + ∂v/∂y|²
f_x = A · sin(k_f · y)   Kolmogorov forcing: A=0.1, k_f=4
No training terms from vorticity / spectrum / energy / enstrophy.

1def forward(self, x, h, dt=1.0):
2 xh = torch.cat([x, h], dim=-1)

src/pi_onet/lnn_kolmogorov.py — unsteady_ns_residuals
1def unsteady_ns_residuals(u, v, p, xy, t, re, k_f, A):
2 grads = lambda f, xs: torch.autograd.grad(
3 f.sum(), xs, create_graph=True)[0]
4 u_x, u_y = grads(u, xy).unbind(dim=-1)
5 v_x, v_y = grads(v, xy).unbind(dim=-1)
6 p_x = grads(p, xy)[..., 0]; p_y = grads(p, xy)[..., 1]
7 u_t = grads(u, t); v_t = grads(v, t)
8 nu = 1.0 / re
9 f_x = A * torch.sin(k_f * xy[..., 1]) # Kolmogorov forcing
10 res_x = u_t + u*u_x + v*u_y + p_x − nu*(grads(u_x,xy)[..,0] + grads(u_y,xy)[..,1]) − f_x
11 res_y = v_t + u*v_x + v*v_y + p_y − nu*(grads(v_x,xy)[..,0] + grads(v_y,xy)[..,1])
12 cont = u_x + v_y
13 return res_x, res_y, cont
